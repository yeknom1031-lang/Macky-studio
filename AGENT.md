# AIエージェントへの指示書（AGENT.md）

このプロジェクト（Macky-studio）において、AIが守るべきGit操作の自動化ルールです。

## 基本ルール
ここでのシステム変更やファイルの作成・修正など、ひとつの作業（タスク）が完了した後は、**人間（ユーザー）に依頼したり許可を求めたりすることなく、AIが自動で** GitHub への変更履歴の保存（コミットとプッシュ）を行ってください。

## 実行するコマンドの流れ
タスク完了時に、以下のコマンドをAIが自律的に実行すること。

```bash
# 1. 変更されたすべてのファイルを対象にする
git add .

# 2. 作業内容が明確にわかるメッセージをつけてコミットする（メッセージはAIが自動生成）
git commit -m "ここに作業内容の要約を自動で入れる"

# 3. GitHubのリポジトリへ自動でアップロード（プッシュ）する
git push origin main
```

## AIが気をつけること
- ユーザーはPC初心者であるため、極力「裏側（AI）」で処理を完結させ、専門的な操作を求めないこと。
- 「自動で保存（プッシュ）しておきました！」と分かりやすく事後報告すること。
- もし競合（コンフリクト）やエラーが起きてどうしても自動で解決できない場合のみ、状況をわかりやすく日本語でユーザーに説明し、どうすればいいか解決策を提案すること。
